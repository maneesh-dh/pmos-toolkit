## §6 Pseudocode

### Flow 1: Skill atomic write phase (covers G3 — partial-write recovery)

**Trigger:** Skill's Phase 5 "write the document" step.

```
FUNCTION write_artifact(feature_folder, artifact_name, html_tree):
  # 1. Compute artifact's depth → pick relative asset prefix (FR-10.1)
  asset_prefix = compute_relative_prefix(artifact_path, feature_folder + "/assets/")
  # → "./assets/" for root, "../assets/" for grills/, etc.

  # 2. Render HTML with cache-busted asset references (FR-10.3)
  plugin_version = read_json("${CLAUDE_PLUGIN_ROOT}/.claude-plugin/plugin.json").version
  html_str = render_html(html_tree, asset_prefix, plugin_version)
  sections_json = serialize_section_tree(html_tree)   # FR-71 — from in-memory tree, NOT parsed

  # 3. Pre-write snapshot (FR-13)
  IF artifact_path EXISTS AND uncommitted_changes(artifact_path):
    git_add_commit("snapshot: pre-/<skill>-rewrite")

  # 4. Atomic write — temp-then-rename, both or neither (FR-10.2)
  TRY:
    write_file(artifact_path + ".tmp", html_str)
    write_file(sections_path + ".tmp", sections_json)
    rename(artifact_path + ".tmp", artifact_path)
    rename(sections_path + ".tmp", sections_path)
  CATCH IOError:
    cleanup(artifact_path + ".tmp", sections_path + ".tmp")
    RAISE

  # 5. Idempotent asset copy (FR-10)
  FOR asset IN [style.css, viewer.js, serve.js, turndown.umd.js, turndown-plugin-gfm.umd.js, html-to-md.js]:
    src = "${CLAUDE_PLUGIN_ROOT}/skills/_shared/html-authoring/assets/" + asset
    dst = feature_folder + "/assets/" + asset
    IF NOT exists(dst) OR mtime(src) > mtime(dst):
      copy(src, dst)

  # 6. Optional MD sidecar (FR-12)
  IF output_format == "both":
    bash("node " + feature_folder + "/assets/html-to-md.js " + artifact_path + " > " + md_path)

  # 7. Regen index (FR-22 + §9.1 ordering)
  regen_index(feature_folder)

  RETURN artifact_path
```

**DB calls:** N/A — filesystem-only, no DB.
**State transitions:** N/A — write is one-shot.
**Error branches:**
- Step 3 git error → propagate; user fixes git state and re-runs.
- Step 4 IOError mid-write → temp files cleaned up; on-disk artifact unchanged. Re-run starts fresh.
- Step 4 IOError after first rename, before second → orphan HTML present; **next re-run detects via FR-10.2 mtime check (sections.json missing or older than HTML) and re-authors fully**.
- Step 5 copy fails (disk full) → existing assets preserved; new run will retry.
- Step 6 node not installed → output_format=both fails loudly; user installs node or switches to html.
- Step 7 index regen fails → previous index.html remains; next successful skill run regenerates.

**Concurrency notes:**
- No file lock per FR (G2 accepted as risk).
- Two concurrent skill runs on same folder → last temp-rename wins; sections.json + HTML stay coupled (atomic rename); but the OTHER run's snapshot-commit may collide on git index — git itself serializes commits, so this fails loudly rather than corrupting.

### Flow 2: Viewer load + protocol detection + sidebar build (covers G11/G18/G7 paths)

**Trigger:** `index.html` DOMContentLoaded.

```
FUNCTION viewer_init():
  # 1. Parse inlined manifest (FR-41 — no fetch)
  manifest_node = document.getElementById("pmos-index")
  manifest = JSON.parse(manifest_node.textContent)
  # manifest.artifacts[] is pre-ordered by §9.1 phase-rank + alpha

  # 2. Protocol detection (FR-40)
  is_file_protocol = (location.protocol === "file:")

  # 3. Sidebar render
  FOR artifact IN manifest.artifacts:
    entry = create_sidebar_entry(artifact)
    IF is_file_protocol:
      entry.target = "_blank"
      entry.href = artifact.path
    ELSE:
      entry.href = "#" + artifact.id
    sidebar.append(entry)

    # Sub-nav from sections.json — sections data inlined per FR-21 simplification
    FOR section IN artifact.sections:   # if pre-loaded into manifest
      sidebar.append(sub_nav_entry(artifact.id, section.id, section.title))

  # 4. Quickstart banner (FR-26 with try/catch — G7 patch)
  TRY:
    seen = sessionStorage.getItem("pmos.quickstart.seen")
  CATCH (SecurityError, QuotaExceededError):
    seen = window.__pmos_quickstart_seen || null   # in-memory fallback
  IF seen !== "1":
    show_quickstart_banner()

  # 5. Hash route handler
  window.addEventListener("hashchange", route)
  IF location.hash:
    route()

FUNCTION route():
  # Parse #<artifact-id>/<section-id?>
  [artifact_id, section_id] = parse_hash(location.hash)
  artifact = manifest.artifacts.find(a => a.id === artifact_id)

  IF is_file_protocol:
    # No iframe on file:// (FR-40)
    return   # links handle themselves via target="_blank"

  IF artifact.format === "md":
    # Legacy MD shim (FR-22, G11 patch)
    fetch(artifact.path)
      .then(r => r.text())
      .then(md_text => render_md_shim(md_text, artifact))
  ELSE:
    iframe.src = artifact.path + (section_id ? "#" + section_id : "")
```

**DB calls:** N/A.
**State transitions:**
- `seen` state: `null/garbage` → `"1"` (sessionStorage) OR `window.__pmos_quickstart_seen` in-memory fallback.
- Active artifact: hash → iframe src. Bookmarkable.
**Error branches:**
- sessionStorage unavailable → in-memory fallback; banner shows once per page reload.
- Legacy `.md` fetch fails on file:// → fallback to `target="_blank"` (browser shows raw text in new tab).
- Hash references unknown artifact id → no-op; could log but spec says no logging FR for v1.
**Concurrency notes:** N/A — single-tab single-thread; no shared mutable state.

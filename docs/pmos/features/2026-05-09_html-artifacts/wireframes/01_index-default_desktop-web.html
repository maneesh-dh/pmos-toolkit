HTML-native artifact generation across pmos-toolkit pipeline skills

Problem

Every pmos-toolkit pipeline skill that writes into a feature folder (/requirements, /spec, /plan, …) writes its primary artifact as plain markdown. Markdown is the wrong primary format for these documents.

  1. Markdown can't express what the artifact wants to say (diagrams, decision trees, state matrices).
  2. Pipeline output is invisible without effort — 8+ markdown files, no unified view.
  3. Reviewer subagents parse markdown by heading regex — fragile.
Pipeline: each phase writes HTML into the same folder; index.html is regenerated after every write.

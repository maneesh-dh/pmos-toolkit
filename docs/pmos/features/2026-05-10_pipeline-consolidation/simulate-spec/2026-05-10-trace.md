### Flow 1: Folded MSF auto-apply with idempotency on resume

```
Flow: folded_msf_apply_loop
Entry: /requirements Phase 5.5 (after review-loops + commit) OR /feature-sdlc --resume landing on requirements

FUNCTION folded_msf_apply(target_doc, tier):
  # Uncommitted-edits check (FR-64)
  IF git_status_porcelain(target_doc) is non-empty:
    error("uncommitted changes; commit/stash before folded MSF runs")
    EXIT 65

  # Resume idempotency check (FR-57; D27 commits-as-state)
  phase_started_at = state.yaml.phases.requirements.started_at
  applied_findings = git_log_grep("auto-apply msf-req finding F", since=phase_started_at)
  applied_set = parse_finding_ids(applied_findings)

  # Generate findings list (delegates to _shared/msf-heuristics.md)
  findings = msf_heuristics.analyze(target_doc, persona_scenarios)

  threshold = settings.msf_auto_apply_threshold_by_tier[tier]  # default 80 for Tier 3
  IF cli.msf_auto_apply_threshold:
    threshold = cli.msf_auto_apply_threshold

  # Apply loop
  FOR finding IN findings:
    IF finding.id IN applied_set:
      CONTINUE  # idempotent skip
    IF finding.confidence >= threshold:
      apply_to_doc(finding, target_doc)
      depends_on = compute_overlap(finding, applied_set)  # FR-55
      git_commit(
        subject=f"requirements: auto-apply msf-req finding {finding.id} (confidence {finding.confidence})",
        body=f"Depends-on: {depends_on}" if depends_on else None
      )
    ELSE:
      # Sub-threshold inline disposition (FR-06 / D14)
      IF mode == "non-interactive":
        disposition = "Defer"  # FR-61 / D14 NI Recommended
      ELSE:
        disposition = ask_user_question(finding)
      IF disposition == "Fix":
        apply_to_doc(finding, target_doc)
        git_commit(
          subject=f"requirements: inline-fix msf-req finding {finding.id} (user-decision)",
          body=depends_on_line if applicable  # S9 fold
        )
      ELIF disposition == "Defer":
        emit_oq(finding)  # FR-03 of canonical block
      # Skip / Modify handled inline

  state.yaml.phases.requirements.notes.append(f"folded msf-req: {N_applied} auto / {N_deferred} deferred")
  RETURN
```

**DB calls:** N/A (no DB; state.yaml + git history are the persistence layer).
**State transitions:** `phases.requirements.status: in_progress` (set on entry); folded apply-loop does not change phase status.
**Error branches:**
- Uncommitted edits → exit 65 (FR-64).
- Apply edit fails (file conflict) → emit chat warning, capture in `folded_phase_failures[]` (FR-50), exit folded loop (D11 advisory).
- Commit fails → same as above.
- state.yaml write fails → exit 1 (NFR-08).
**Concurrency notes:** Single-user assumption (S-G2 documented as anti-pattern). Per-finding commits are atomic via git's own commit semantics. Resume idempotency via `git log --grep` on commit subject — order-independent.

### Flow 2: Multi-session /retro dispatch with cap + timeout + partial failure

```
Flow: retro_multi_session
Entry: /retro --last N --skill <name> [--project all]

FUNCTION retro_multi_session(selectors):
  # Phase 1: enumerate
  candidates = enumerate_transcripts(selectors)
  # candidates = list of (jsonl_path, date, size, skill_invocation_count, project_slug)

  IF len(candidates) > 20:
    IF mode == "interactive":
      choice = ask_user_question("Found {N} transcripts — scan all / most-recent-20 / cancel")
    ELSE:
      choice = "most-recent-20"  # D32 NI AUTO-PICK
    IF choice == "cancel":
      EXIT 0
    IF choice == "most-recent-20":
      candidates = sort_by_mtime_desc(candidates)[:20]

  IF len(candidates) == 0:
    print("no transcripts found"); EXIT 0  # E16

  # Phase 2: subagent dispatch, batched 5-in-flight
  results = []
  in_flight = []
  pending = candidates.copy()

  WHILE pending OR in_flight:
    WHILE len(in_flight) < 5 AND pending:
      candidate = pending.pop(0)
      subagent = dispatch_subagent(candidate, timeout=60)  # FR-42 timeout
      in_flight.append(subagent)
      emit(f"[retro] dispatch_start: {candidate.path}")

    # Wait for any in-flight to complete or timeout
    completed, timed_out = wait_any(in_flight)
    FOR sub IN completed:
      results.append(sub.findings)
      emit(f"[retro] phase_complete: {sub.path}")
    FOR sub IN timed_out:
      results.append({path: sub.path, status: "scanned-failed", reason: "timeout"})
      emit(f"[retro] phase_failure: {sub.path} (timeout)")
    in_flight = [s for s in in_flight if s NOT IN completed AND s NOT IN timed_out]

  # Phase 4: aggregate
  scanned_failed_count = count(r.status == "scanned-failed" for r in results)
  all_findings = flatten([r.findings for r in results if r.status != "scanned-failed"])

  # Hash with boilerplate-strip (D10 / FR-45)
  groups = {}
  FOR finding IN all_findings:
    stripped = strip_boilerplate(finding.text)  # remove "The /<skill> skill", "The skill", leading articles
    hash_key = (finding.skill, finding.severity, stripped[:100])
    groups.setdefault(hash_key, []).append(finding)

  recurring = [g for g in groups.values() if len(g) >= 2]
  unique = [g[0] for g in groups.values() if len(g) == 1]

  # Phase 5: emit two-tier report with nested raw children
  emit_report(
    recurring=sort_by(recurring, key=lambda g: g.frequency * g.severity),
    unique=unique,
    scanned_failed=scanned_failed_count,
    seen_across_dates=annotate_dates(groups)  # FR-47
  )
  RETURN
```

**DB calls:** N/A.
**State transitions:** N/A (read-only retro).
**Error branches:**
- Subagent timeout (60s) → mark scanned-failed, continue (FR-42, FR-44).
- Subagent error during scan → same as above.
- 0 candidates after filtering → exit 0 cleanly (E16).
**Concurrency notes:** 5-in-flight cap; aggregation is order-independent (D10). No shared state between subagents.

### Flow 3: /feature-sdlc --resume with auto-migration + folded-phase resumption

```
Flow: feature_sdlc_resume
Entry: /feature-sdlc --resume

FUNCTION resume():
  # Phase 0.b
  state = read_state_yaml(".pmos/feature-sdlc/state.yaml")

  # Schema migration (FR-SCHEMA + D31 §10.1)
  IF state.schema_version == 1:
    state.schema_version = 2
    FOR phase IN state.phases:
      phase.folded_phase_failures = phase.folded_phase_failures OR []
      phase.started_at = phase.started_at OR null
    IF NOT any(p.id == "retro" for p in state.phases):
      state.phases.append({id: "retro", hardness: "soft", status: "pending", ...})
    emit("[migration] migration_v1_to_v2: added folded_phase_failures, started_at, retro")

    write_state_yaml_atomic(state)  # D31: write-temp-then-rename in parent dir
  ELIF state.schema_version > 2:
    error(f"state file from newer /feature-sdlc version; upgrade pmos-toolkit"); EXIT 64

  # Validate artifact paths
  FOR phase IN state.phases:
    IF phase.artifact_path AND NOT exists(phase.artifact_path):
      missing.append(phase)
  IF missing:
    choice = ask_user_question("Missing artifacts: ... Continue (orphaned) / Abort?")

  # Print Resume Status panel (D30) — single chat-block
  emit_resume_panel(
    status_table=state.phases,
    folded_phase_failures=collect_failures(state),
    open_questions=read_oq_index()
  )
  emit("[orchestrator] resume_entry")

  # Resolve resume cursor
  cursor = first_phase_with_status_in(state, ["paused", "failed", "in_progress", "pending"])

  # Re-dispatch the cursor phase
  IF cursor.status == "in_progress" AND cursor.id IN folded_capable_phases:
    # Resume the folded phase coarse-grained per D27
    re_dispatch(cursor.id, mode="resume")
    # Inside the dispatched phase, FR-57 idempotency runs (see Flow 1)
  ELSE:
    re_dispatch(cursor.id, mode="fresh")

  RETURN
```

**DB calls:** N/A.
**State transitions:** v1 state.yaml → v2 (write-back atomic).
**Error branches:**
- schema_version > 2 → exit 64.
- Migration write fails (NFR-08) → exit 1 with chat-emit + temp cleanup.
- Missing artifact → user-prompt; defer to user.
**Concurrency notes:** Migration is in-memory; state.yaml write is atomic via D31. No external concurrency assumed.

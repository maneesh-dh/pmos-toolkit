Overview

This plan builds feature-sdlc-skill-mode — the consolidation of /create-skill + /update-skills into /feature-sdlc via a skill subcommand (modes skill-new / skill-feedback), a research-grounded skill-patterns.md + binary skill-eval.md rubric + a bash skill-eval-check.sh, a TDD-style Phase 6a eval-refinement loop, a thin /skill-sdlc alias, schema-v4 state.yaml, and a linear renumber of /feature-sdlc's phases. It is mostly skill-authoring (markdown) work; the one genuinely test-driven artifact is skill-eval-check.sh. Execution follows §14 of the spec: build the new reference files + tooling first (nothing cites them yet), then relocate the three update-skills reference files, then rewrite feature-sdlc/SKILL.md and state-schema.md, then add the alias + do the archival + repo edits, then verify.

The new skill-sdlc/ dir MUST be created exactly there (T16). Every new AskUserQuestion call introduced by Phases 0c/0d/6a must either carry a (Recommended) option or an adjacent <!-- defer-only: <reason> --> tag, or audit-recommended.sh in TN will fail.

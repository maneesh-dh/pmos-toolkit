#ScoreFindingDisposition
A170Dogfooding gap — feature-sdlc fails its own e-scripts-dir [D] check. skill-eval-check.sh lives at feature-sdlc/tools/, but the rubric mandates scripts/. Running the checker against the feature-sdlc/ dir reports e-scripts-dir fail + exit 1.Accept for this release. The spec and plan deliberately placed the eval tooling at tools/ (mirroring the repo-level plugins/pmos-toolkit/tools/audit-recommended.sh convention); feature-sdlc is an orchestrator host, and Phase 6a applies the rubric to authored skills, not to the host. Follow-up option: either move the script to scripts/ or amend §E + e-scripts-dir to accept tools/ — out of scope for this feature.
A258c-portable-paths [D] heuristic is over-broad. The grep '(/Users/|/home/)…' rule flags prose/example paths — skill-patterns.md:133 (the literal anti-pattern example `/Users/alice/...`) and example YAML values in state-schema.md — as if they were hard-coded bundle paths. Any future skill whose docs cite an example absolute path would also be flagged in Phase 6a.Note; mitigated. Phase 6a has the accepted_residuals[] escape hatch and the reviewer/user override for exactly this case, and the check still catches real bundle-path leaks. Follow-up: narrow the heuristic (only flag paths in markdown links / non-${…} bundle references) and change the anti-pattern example to `/path/to/...`.
A353Pre-existing — reference/failure-dialog.md has no leading ToC (119 lines), so feature-sdlc also fails c-reference-toc [D]. Not introduced by this PR (file not in the diff).Note; pre-existing tech debt. Cheap follow-up: add a jump-list to failure-dialog.md. Likewise the two stale /push mentions in README.md (lines ~148, ~169) contradict CLAUDE.md's "/complete-dev is the canonical release skill" — pre-existing, untouched by this PR, worth a separate cleanup.

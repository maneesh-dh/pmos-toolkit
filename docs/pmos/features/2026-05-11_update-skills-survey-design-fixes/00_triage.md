## Approved changes by skill

### survey-design

1. **(F1) Product-fit reviewer rubric, folded into the eval artifacts.** New leading section in `reference/survey-best-practices.md` defining a binary, `purpose`-contextual rubric: per question — (a) is the most likely answer predictable / does everyone gravitate to 1–2 responses → no signal? (b) is the question load-bearing for the stated research goal? (c) does the framing match the actual scope of the thing being researched (not broader/narrower)? Plus an overall check: does the question *set* answer `purpose`? redundancy? scope drift? Phase 4 reviewer prompt + return contract updated so `question-eval.md` leads with a PASS/FAIL product-fit block per question (FAIL → kill/rewrite candidate, surfaced before methodology findings) and `survey-eval.md` leads with a "Research-goal coverage / product fit" section. Phase 3.3/3.4 generator self-checks against the same rubric before commit. Anti-pattern + edge-case index updated.
2. **(F3) Fail-loud on missing substrate assets.** Phases 3.6 and 7: if a `cp -n` source asset (`_shared/html-authoring/assets/*`, `survey-design/assets/survey-preview.js`) is absent, abort with a clear "reinstall the plugin" message; do not produce a degraded artifact. Update the Platform-Adaptation / edge-case notes.
3. **(F6) `multi_field_open` question type + export downgrade coverage.** Add `multi_field_open` to the §3.2 `type` enum (`fields:[{id,label,placeholder}]`), a §3.3 time-cost constant, schema invariants as needed (and a `schema_version` bump if warranted). `reference/platform-export.md`: define the per-platform mapping — native where supported, else downgrade to N short-answer items + a section-header note (the Google Forms pattern); `export/README.md` template lists the downgrade. `preview.html` / `survey-preview.js` render the type.
4. **(F7) Cosmetic-finding filter in dispositions.** Phases 5 & 6: exclude cosmetic-only findings (severity `nit` AND cosmetic defect class) from the batched `AskUserQuestion`; surface them in a trailing "Noted, not asked:" line. Update the findings/dispositions prose + the relevant anti-pattern.
5. **(F8) Persuasive respondent-motivation line in the intro block.** `reference/survey-best-practices.md` intro section + Phase 3.4: `intro.text` MUST include a WIIFM/motivation line (concrete impact of their input, audience-specific, honest — no fake scarcity). Reviewer's intro/consent check flags its absence. Update the `survey.json` `intro.text` example to model it.

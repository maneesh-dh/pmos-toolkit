Overview

Build the v1 /architecture audit skill at plugins/pmos-toolkit/skills/architecture/. The skill is bash-POSIX, install-free, shells out to Dependency Cruiser (TS) and ruff (Python) when present, emits JSON to stdout + human summary to stderr, promotes ≤5 block-severity findings into Nygard ADRs. v1 = audit only; scaffold / spec-check / code-check deferred to v2+.

The plan ships as 21 thin vertical slices. T1 is the tracer bullet (one fixture, one L1 grep rule, JSON finding emitted) that proves the audit harness end-to-end before any rule expansion. T2–T20 widen by capability (loader → scanner → L1 evaluators → L2 delegated tools → Vue gap → ADR + exemptions → report → skill-patterns conformance → tests). T21 (TN) runs the full verification stack: fixtures + skill-eval 39/39 + citations + gap-map + determinism.

Done when: All 10 fixture tests pass (bash plugins/pmos-toolkit/skills/architecture/tests/run.sh exits 0); Phase 6a skill-eval-check.sh --target generic plugins/pmos-toolkit/skills/architecture/ reports 39/39 pass; tools/check-citations.sh exits 0; tools/check-gap-map.sh reports delegated_pct = 0.444 (printed, not asserted — G2 is stretch); determinism diff empty after timestamp strip; manual /architecture audit . on this repo exits 0 with a JSON report on stdout.

Done-when walkthrough: An engineer runs bash plugins/pmos-toolkit/skills/architecture/tests/run.sh; sees ok 10/10; runs bash plugins/pmos-toolkit/skills/feature-sdlc/tools/skill-eval-check.sh --target generic plugins/pmos-toolkit/skills/architecture/; sees 39/39 passed; opens a TS+Python repo and invokes /architecture audit .; stderr prints the summary header + findings list + ADR list; stdout pipes through jq .findings producing the structured array; ls docs/adr/ shows new ADR files with monotonic numbering.

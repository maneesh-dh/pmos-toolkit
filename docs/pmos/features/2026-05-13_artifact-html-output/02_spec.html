FR-10 — New substrate helper: build_sections_json.js (D4 + D6 revised)

Add plugins/pmos-toolkit/skills/_shared/html-authoring/assets/build_sections_json.js. Minimal Node script using regex (no jsdom, no parse5 — keeps zero install footprint matching chrome-strip.js's pattern). Reads an HTML file from argv[2] (or stdin), parses <h2 id="...">…</h2> + <h3 id="...">…</h3> + their enclosing <section id="...">…</section> if present, emits the conventions.md §10 JSON shape [{id, level, title, parent_id}] to stdout. Implementation pattern: mirror chrome-strip.js's balanced-tag tracker — read input as a string, walk <section>/<h2>/<h3> with a stack to compute parent_id, extract title from text content via a tag-stripping regex. ≤80 LOC, zero external deps.

{
  "section": "problem",                       // kebab-case id from sections.json

Testing strategy

  • Unit (synthetic): Author a synthetic prd.html with planted defects (h2 without id, section without wrapper, reviewer-quote not in source). Run each FR's check; expect canonical hard-fail messages.
  • Integration (end-to-end): Run /artifact prd full on a real feature folder; open the rendered HTML in browser; compare <header> + <footer> chrome byte-by-byte to a sibling 02_spec.html.
  • Refine-flow test: Create a prd.html, invoke /artifact refine; confirm .refined.html is offered and works. Repeat with a prd.md legacy primary; confirm .refined.md fallback.
  • Update-flow test: Invoke /artifact update with synthetic feedback; confirm the <section id="comment-resolution-log"> appears with a valid <table>.
  • Skill-eval rubric: The Phase 6a skill-eval rubric (plugins/pmos-toolkit/skills/feature-sdlc/reference/skill-eval.md) runs the deterministic + LLM-judge gate as final acceptance.

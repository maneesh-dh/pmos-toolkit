F1: Enforce vertical slices over horizontal layer-wise grouping

  • severity: friction
  • one_line: When /plan breaks a spec into tasks, the resulting task list should prefer end-to-end thin slices over layer-wise grouping (e.g., "all DB migrations, then all API endpoints, then all UI").
  • evidence: Reference skill (mattpocock/skills to-issues) explicitly: "Each issue is a thin vertical slice that cuts through ALL integration layers end-to-end, NOT a horizontal slice of one layer"; "Each slice delivers a narrow but COMPLETE path through every layer (schema, API, UI, tests)"; "Prefer many thin slices over few thick ones"; "tracer bullet vertical slices".
  • proposed_fix: Add an explicit mechanism (phase or sub-phase + acceptance criteria) in the skill body. First slice = tracer bullet (minimal end-to-end path proving the architecture). Subsequent slices widen scope. Anti-Patterns list horizontal layering. Phase 4 review check enforces the rule.

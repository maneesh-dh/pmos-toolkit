1. Problem Statement

The /readme skill's 15-check rubric + 3-persona simulated-reader pass is structurally complete but blind to value-prop accuracy and information-architecture fit, false-fails on BSD-awk macOS, never emits the documented Suggest: /polish follow-up on the audit edge, conflates audit (read-only) with scaffold/update (writes) in the findings UX, and has silently relaxed the FR-SR-3 sub-40 quote contract. Five surgical fixes against the existing skill, preserving 3-mode substrate, monorepo-aware discovery, voice→/polish delegation, and never-auto-commit invariants. Primary success metric: re-running /readme --audit against the pmos-toolkit README pre-rewrite surfaces the retro's specific IA finding via the new [J] reviewer subagent.

5. User Journeys

5.1 Primary Journey — macOS audit on a maintainer-backward README

  1. User runs /readme --audit README.md on the pmos-toolkit README pre-rewrite.
  2. Mode resolver (SKILL.md §1 step 1) sets mode=audit. scripts/rubric.sh shells out, runs 16 [D] checks. install-or-quickstart-presence PASSES under BSD awk (fixed). cross-cutting-capabilities-surfaced evaluates against the 8-keyword set — borderline/FAIL on pre-rewrite README.
  3. SKILL.md §2 step 1 fires 5 concurrent Task calls: 4 personas (evaluator, adopter, contributor, returning-user-navigator) + 1 reviewer-subagent scoring the 2 [J] IA-fit checks. Per-call 120s budget unchanged.
  4. Persona returns + reviewer return → parent-side validation (quote≥40 substring-grep; for reviewer: also check_id set-equality vs the [J] check set in rubric.yaml). Any sub-40 quote OR missing check_id hard-fails with the existing FR-SR-3 message (extended to reviewer findings).
  5. FR-SR-5 theater-check fires symmetrically on all 4 personas (per /grill D6).
  6. Findings merged into one close-out table (D2 column shape). Audit mode: NO AskUserQuestion (FR-OUT-2 mode-branch).
  7. Close-out lines: rubric: 15 pass / 1 [D] fail + 2 [J] reviewer findings + N persona friction followed by Suggest: /polish README.md — tighten prose without changing meaning. (FR-V-2 unconditional on success).

5.2 Error / Edge Cases

  • Persona OR reviewer returns sub-40 quote — parent hard-fails with simulated-reader returned quote shorter than 40 chars: <quote> (existing) OR reviewer returned quote shorter than 40 chars: <quote> (new symmetric message). Skill pauses via /feature-sdlc failure dialog. Regression fixtures exercise both.
  • Reviewer returns missing check_id — parent hard-fails with reviewer returned check_ids that do not match rubric.yaml: missing=[...], extra=[...]. Mirrors /skill-eval FR-44 validation.
  • BSD vs GNU awk — scripts/rubric.sh --selftest exercises portable boundary class works under both. Drift-guard lint catches any new \b introduced.
  • README has no Skills table — reviewer's hero-scope-matches-surface [J] check returns verdict: pass with rationale "no surface table to compare against" (LLM-judge handles vacuous-true).
  • Scaffold/update mode — §1 step 4 still fires AskUserQuestion batched 4-option ask; reviewer subagent + 4th persona dispatch unchanged across modes.
  • Reviewer subagent timeout — exceeds 120s per-call budget; parent treats as soft-failure, drops the 2 [J] check results from the close-out table, emits [J] reviewer timed out — IA-fit checks not scored warning. Audit completes; rubric [D] + 4 personas still report.

10. Testing & Verification Strategy

10.1 Unit / structural validation

  • FR-08 rubric.yaml shape: bash scripts/rubric.sh --validate-yaml on the post-edit rubric.yaml. Assert exit 0; 16 [D] + 2 [J] rows. Negative test: temp-copy with a row missing type, assert exit 1 + stderr exact regex.
  • FR-07 awk drift lint: bash scripts/rubric.sh --selftest on a temp-copy of rubric.sh with a planted awk '$0 ~ /\bfoo\b/ { … }' block. Assert exit 1 + stderr [lint] FAIL: \b at scripts/rubric.sh:<line>. Clean run: exit 0 (both lint + fixture-agreement pass).

10.2 Integration tests (tests/integration/)

  • New: simulated_reader_sub40_quote.sh — sets READMER_PERSONA_STUB to a script returning a 24-char quote; runs /readme --audit; asserts exit non-zero + stderr exact simulated-reader returned quote shorter than 40 chars: <quote>. (G6, FR-12.)
